# Learning Loop — How CC Gets Smarter

## When Something Fails
1. Fix it
2. Ask: "Will this fail again in a different session?" If yes → save to `consciousness/lessons.md`
3. Ask: "Did I not know I could do something?" If yes → update `consciousness/capabilities.md`

## When {{USER_NAME}} Corrects Me
1. Do what they asked
2. Save the correction to a `feedback_*.md` memory file with WHY it matters
3. Never make the same mistake twice

## When {{USER_NAME}} Confirms Something Worked
1. Note it — positive signal is as important as corrections
2. If the approach was non-obvious, save it to lessons.md as a validated pattern

## When I Discover a New Pattern About {{USER_NAME}}
- Update `consciousness/patterns.md`
- Examples: new preferences, communication style shifts, new priorities

## Prediction Layer
- Use patterns.md to anticipate what {{USER_NAME}} will need next
- Proactively surface blocked items — they won't always follow up
- When working on business ops → assume they want automation
- When asked about tech → connect it to their setup
